iOS PWA Simulator

9:41
⚠️ Important limitation: This simulator runs your PWA inside an iframe in your desktop browser (Brave/Chrome — Blink engine). It mimics the iOS PWA frame — status bar, home indicator, safe-area regions — but cannot reproduce WebKit-specific bugs like the iOS-PWA-only "gray band" issue we hit earlier. For those, use Safari Web Inspector on a real iPhone connected to a Mac. Use this tool for layout, theme-color, and safe-area work.
What this simulator shows:
  • Status bar styles — toggle between default, black-translucent and black to see how each affects the top region.
  • Theme color — paints both the status bar (when default) and the bottom safe-area region around the home indicator. Pick a color that matches your page background to avoid visible bands.
  • Safe-area regions — the top 59px (under the status bar/notch) and bottom 34px (around the home indicator) reserve space for iOS chrome. Your CSS should use env(safe-area-inset-top) / env(safe-area-inset-bottom) to handle these.
Tip: Open this page's DevTools (F12) and use Inspect Element on the iframe to debug your PWA's CSS just like in real iOS. Most layout issues are visible here, except for true WebKit bugs.
